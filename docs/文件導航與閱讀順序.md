# 文件導航與閱讀順序

## 這份文件的用途
這份文件是給未來的新對話、新助理，或隔一段時間回來的自己看的。目的是在最短時間內恢復上下文，避免每次都重新從零理解整個專案。

## 若你只看 3 份文件
最優先看：
1. `docs/目標與方向.md`
2. `docs/變更與進度.md`
3. `docs/pipeline_architecture.md`

看完這三份，通常就能理解：
- 專案要做什麼
- 目前做到哪裡
- 主架構大致長什麼樣

## 若你要接手實作工作
請按這個順序看：

### 第 1 層：先理解全局
1. `docs/目標與方向.md`
2. `docs/設計方案.md`
3. `docs/變更與進度.md`

### 第 2 層：理解 execution 主線
4. `docs/pipeline_architecture.md`
5. `docs/ExecutionController_v1_實作說明.md`

### 第 3 層：理解合作方式與流程
6. `docs/合作與貢獻指南.md`
7. `docs/工作流程與Continue設定.md`

### 第 4 層：補充歷史與規範
8. `docs/CHANGELOG_變更紀錄.md`
9. `docs/CONTRIBUTING_開發指南.md`

## 若你要碰高風險 execution 檔案
除了上面文件，還應直接看程式與測試：
- `repo_guardian_mcp/services/execution_controller.py`
- `repo_guardian_mcp/services/edit_execution_orchestrator.py`
- `repo_guardian_mcp/services/task_orchestrator.py`
- `repo_guardian_mcp/tools/run_task_pipeline.py`
- `tests/test_execution_controller.py`
- `tests/test_execution_controller_v1.py`
- `tests/test_run_task_pipeline.py`
- `tests/test_validation_pipeline.py`
- `tests/test_rollback_session.py`

原因：這些檔案共同定義了真實 execution contract。

## 若你只想知道目前最新狀態
請直接讀：
- `docs/變更與進度.md`
- `docs/CHANGELOG_變更紀錄.md`

這兩份應該要能回答：
- 最近一輪做了什麼
- 現在穩定基準是什麼
- 下一步建議做什麼

## 若你想知道為什麼不能亂改 controller
請讀：
- `docs/ExecutionController_v1_實作說明.md`
- `docs/pipeline_architecture.md`

## 對未來新助理的快速建議
如果你是新對話中的助理，建議用這個最短路徑：
1. 先看 `docs/目標與方向.md`
2. 再看 `docs/變更與進度.md`
3. 再看 `docs/pipeline_architecture.md`
4. 若要碰 execution，補看 `docs/ExecutionController_v1_實作說明.md`
5. 之後再看程式碼與測試
