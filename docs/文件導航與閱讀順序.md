# 文件導航與閱讀順序

## 這份文件的用途
這份文件是給未來的新對話、新助理，或隔一段時間回來的自己看的。  
目的不是只讓人知道「有哪些文件」，而是能在最短時間恢復這個專案的**真正定位**：

> **這個專案最終要成為本地端 CLI Agent（具備 Skill 系統），並以可媲美 Cursor Agent 為方向。**

---

## 若你只看 3 份文件
最優先看：

1. `docs/目標與方向.md`
2. `docs/變更與進度.md`
3. `docs/pipeline_architecture.md`

看完這三份，通常就能理解：
- 專案最終要成為什麼
- 目前做到哪裡
- 現有 working baseline 與過渡架構是什麼

---

## 閱讀時要記住的判斷原則
讀文件時，不要只問「現在怎麼運作」，還要問：

- 這是不是最終架構？
- 這是穩定基準、過渡相容層，還是未來方向？
- 這個設計是否服務於 CLI agent + skill system 的終局？
