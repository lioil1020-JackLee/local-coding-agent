## 目前專案的主要目錄角色

### `repo_guardian_mcp/`
這是目前真正的主產品路徑。

它包含：

- CLI 入口
- skill registry
- chat mode
- execution controller
- orchestrator 與 runtime
- session / sandbox / rollback / validation 等服務

### `repo_guardian_agent/`
這個目錄仍存在，但目前結構較小，主要可看到：

- `planner.py`
- `patch_generator.py`
- `validation_controller.py`

它不是目前 `pyproject.toml` 所列出的 package 主線，因此不應再把它描述成「目前主要對外執行核心」。

### `docs/`
目前 docs 已整併成四份核心文件，目的不是做歷史碎片備忘錄，而是作為：

- 架構說明
- 設計決策脈絡
- handoff 材料
- 開發與維護入口

### `tests/`
正式測試目錄。不要只看 README 的口頭描述，判斷是否對齊現況時，應回到 `tests/` 實際內容。

## 高風險誤解區

### 誤解 1：這只是 pipeline 專案
不對。  
目前程式碼已經明顯包含 CLI、chat、session-aware routing、skill registry、controller、orchestrator、trace summary 等更完整的 agent runtime 元件。

### 誤解 2：`repo_guardian_agent` 是主要執行路徑
不對。  
從 `pyproject.toml` 的 package 與 script 設定來看，目前主線是 `repo_guardian_mcp`。

### 誤解 3：skill system 只有 v2，沒有其他演進
也不精確。  
更合理的描述是：目前已具備 v2 metadata 基礎，同時在 chat/help 與程式能力上已朝更完整的下一階段演進。
