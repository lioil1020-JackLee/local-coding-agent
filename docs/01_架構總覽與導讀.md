## 這份文件的用途

本文件幫助新加入的開發者、未來的自己，或新的 AI 助理，在最短時間內理解：

- 目前專案真正的封裝與執行主線
- 哪些目錄是主要產品路徑
- 哪些模組是高風險區
- 文件該怎麼讀，才不會被歷史描述誤導

---

## 一句話心智模型

目前最準確的心智模型是：

> **`local-coding-agent` 是一個以 `repo_guardian_mcp` 為封裝與 CLI 主線的本地端 coding agent 專案。**

repo 中仍保留 `repo_guardian_agent/`，但依目前封裝設定，它不是主 CLI 入口。

## 目前專案的主要目錄角色

### `repo_guardian_mcp/`
這是目前真正的主產品路徑。

它包含：

- CLI 入口
- skill registry
- chat mode
- execution controller
- orchestrator 與 runtime services
- session / rollback / sandbox / validation / trace summary 等服務

### `repo_guardian_agent/`
此目錄仍存在於 repo 中，代表專案保留了另一組 agent 相關模組；但從目前 `pyproject.toml` 與 CLI script 來看，它不是對外封裝主線。

### `docs/`
目前 docs 已整併成四份核心文件，目的是提供：

- 架構說明
- 設計決策脈絡
- handoff 材料
- 開發與維護入口

### `tests/`
正式測試目錄。判斷文件是否對齊現況時，不要只看舊描述，應回到 `tests/` 實際內容。

## 高風險誤解區

### 誤解 1：這只是單一 pipeline 專案
不精確。  
目前代碼已經包含 CLI、chat、skill registry、session-aware interaction、controller、orchestrator、trace summary 等更完整的 agent runtime 元件。

### 誤解 2：`repo_guardian_agent` 是目前主要執行路徑
不對。  
目前 package 與 CLI script 都指向 `repo_guardian_mcp`。

### 誤解 3：manifest 還不存在
不對。  
目前 `skills.py` 已有 `SkillManifest`、`from_json_file()`、`register_manifest()` 等基礎能力。
