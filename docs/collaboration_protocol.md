# Collaboration Protocol

## 這份文件是什麼
這份是我們合作方式的固定說明。
未來如果換新對話，請優先遵守這份方式。

## 使用者背景（固定前提）
- 使用者不會寫程式
- 但有邏輯概念
- 想用白話文下指令
- 不想浪費時間在一堆工程術語上
- 希望直接改檔、直接測試、直接往下一步走

## 合作方式（固定）
### 1. 盡量直接改檔
不要只講抽象概念。
如果已經能定位到檔案，就直接修改並提供下載連結。

### 2. 一次只動一個檔案或一小組檔案
不要一次動太多地方，避免難以測試與回溯。

### 3. 改完後直接告訴使用者怎麼測
例如：
- 覆蓋哪個檔案
- 執行哪個指令
- 預期看到什麼結果

### 4. 測試通過再進下一步
不要同時開很多戰線。

### 5. 一律使用繁體中文
而且盡量用白話文。

### 6. 程式碼註解一律用繁體中文
這是固定原則，不要跳掉。

### 7. 分析與修改要嚴格分開
分析時不能改檔。
修改時不能把聊天說明寫進程式。

## 理想合作節奏
1. 使用者提出目標
2. 先定位最該改的檔案
3. 直接修改
4. 提供下載連結
5. 使用者覆蓋回本地
6. 使用者執行測試
7. 回報結果
8. 再做下一步

## 新對話中建議的起手式
如果要快速接手，建議先講：
- 目前已完成到哪裡
- 現在卡在哪一個檔案或哪一個流程
- 想先處理哪一個問題
- 請先讀 `docs/local_coding_agent_master_plan.md`
- 請遵守 `docs/collaboration_protocol.md`

## 我們目前共同接受的產品目標
我們不是在做一個只靠 prompt 的小工具。
我們要做的是：

**本地端、正式可用、長期維護、接近 Cursor 體驗的 coding agent**

而且一定要符合：
- 新手友善
- 繁體中文
- 白話指令
- 能自己找檔案與找入口點
- 分析不改檔
- 修改可驗證、可回滾
