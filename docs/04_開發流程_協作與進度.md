# 04_開發流程_協作與進度

## 這份文件在做什麼

這份是「實際執行表」。
你每天要做什麼、每週怎麼推進、如何確認沒有走偏。

---

## 日常流程（單人本地）

### 上工前（5 分鐘）

1. 跑 `ops run --profile day-start`
2. 看 preflight 與 health 結果
3. 若 Continue 設定漂移，先 sync

### 開發中

1. 先 plan，再 run
2. 修改任務一定看 diff + validation
3. 出錯先看 trace，再決定 rollback

### 收工前（5 分鐘）

1. 跑 `ops run --profile day-end`
2. 產生 snapshot
3. 執行 runtime cleanup（至少 dry-run）

---

## 每週節奏（建議）

1. 週一：確認本週任務集與驗收門檻
2. 週中：補齊測試缺口、修正高頻錯誤類型
3. 週末：跑完整 E2E 與 benchmark，更新文件

---

## 交付定義（Definition of Done）

一個功能要算完成，至少要同時滿足：

1. 有對應測試（單元或整合）
2. CLI 輸出符合 envelope 契約
3. 有 trace 可追蹤關鍵步驟
4. README + docs 至少一處同步更新
5. 不破壞既有主線命令

---

## 建議固定檢查命令

```bash
uv run pytest -q
uv run repo-guardian continue-e2e run .
uv run repo-guardian benchmark init .
uv run repo-guardian benchmark run . --threshold 0.85
uv run repo-guardian benchmark run . --tasks-file agent_runtime/benchmarks/corpus.v1.json --threshold 0.85
uv run repo-guardian health report .
uv run repo-guardian ops run . --profile release-check --continue-source-config continue/config.yaml --continue-target-config F:\.continue\config.yaml
```
