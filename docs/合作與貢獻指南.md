# 合作與貢獻指南

本文件描述專案的協作模式與貢獻規則，確保參與者能安全且高效地推進開發。  

## 合作模式

為確保安全與效率，請遵循以下合作流程：

- 助理（Agent）完成修改後，會提供更新的檔案內容與路徑，通常以壓縮檔形式提供，避免零散檔案。  
- 使用者下載並覆蓋本地檔案後，執行 `uv run pytest -q` 以驗證變更是否通過所有測試。  
- 若變更涉及多輪指令，助理會提供 `conversation_id` 與 `session_id` 以便後續追蹤。  
- 助理會告知使用者本次使用的工具流程，例如 `run_task_pipeline`、手動 session 工作流或高階 refactor 工具。  
- 在交流過程中請使用繁體中文，並盡可能用白話來描述需求。  

## 基準與狀態

目前專案基準如下：

- pytest 測試：18 項全部通過。  
- MCP server 骨架可啟動並支援 API 金鑰認證與日誌。  
- `run_task_pipeline`、`create_task_session`、`propose_patch`、`preview_diff`、`stage_patch`、`run_validation_pipeline`、`move_file` 等工具可正常運作。  
- 初步整合 Planner、Patch Generator、Validation Controller，可解讀模糊指令並生成初步補丁。  
- conversation 記憶機制已支援 `conversation_id`，將逐步擴充為多輪對話與長期任務。  

## 開發流程

參與專案開發時，請遵循以下流程：

1. **執行測試**：啟動虛擬環境後執行 `uv run pytest -q` 確認所有測試通過，新增功能時請補充 pytest 測試。  
2. **啟動 MCP server**：設定環境變數 `MCP_API_KEY`，使用 `uvicorn repo_guardian_mcp.server:app --reload` 啟動服務，可透過 HTTP 或 Continue 進行調用。  
3. **遵循安全編輯**：所有修改需在 sandbox 執行並通過驗證；不得直接將聊天內容或說明文字寫入程式碼。  
4. **撰寫繁體中文註解**：程式內的註解及溝通說明均使用繁體中文，保持簡潔、與程式邏輯分離。  
5. **執行 lint 與型別檢查**：使用 `black`、`flake8`、`mypy` 等工具保持程式風格一致與型別正確。  
6. **更新文件與 Changelog**：任何新功能或重大調整都需更新對應的說明文件與變更紀錄。  
7. **送出貢獻前檢查**：確認所有測試、lint、型別檢查均通過，md 文件已同步更新。  

## 貢獻規則

- **維持測試基準**：不得移除或忽略既有測試，除非重構並提供替代測試。  
- **向下相容性**：避免隨意修改工具介面與檔案格式，除非經充分討論並公告。  
- **避免臨時 Patch**：以長期可維護的架構設計為主，謹慎處理權宜之計。  
- **尊重協作流程**：遵循安全編輯與驗證機制，確保每次提交都是穩定且可追溯的。
