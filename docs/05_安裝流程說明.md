# 05_安裝流程說明（從零到可用）

## 這份文件給誰看

給「幾乎不會寫程式、剛接觸本地 AI agent」的人。
你只要照順序做，不需要先懂工程術語。

---

## 你的最終目標

做完這份文件後，你會有：

1. 一個可在本機跑的 AI coding agent
2. Continue 可以正常聊天、分析、修改
3. 支援網路查詢（Tavily + DDG 備援）
4. 可接 LM Studio 本地模型（Qwen2.5 / Qwen3）

---

## 全流程總覽（先看一次）

你會做 12 件事：

1. 安裝 Git
2. 安裝 Python 3.12
3. 安裝 Node.js（含 npm）
4. 安裝 VS Code
5. 安裝 Continue 擴充套件
6. 安裝 LM Studio
7. 下載本專案
8. 安裝專案依賴（uv sync）
9. 設定 Continue 資料夾路徑
10. 設定 API 環境變數（Tavily / GitHub）
11. 啟動 LM Studio 模型與 API
12. 最後驗證（測試 + MCP + Continue）

---

## 第 1 步：安裝 Git

1. 到 Git 官網下載 Windows 安裝檔  
   `https://git-scm.com/download/win`
2. 一路按 Next（預設選項即可）
3. 安裝完成後，開啟 PowerShell，輸入：

```powershell
git --version
```

看到版本號就代表成功。

---

## 第 2 步：安裝 Python 3.12

1. 到 Python 官網下載 3.12（Windows x64）  
   `https://www.python.org/downloads/`
2. 安裝時一定要勾：
   `Add python.exe to PATH`
3. 安裝完成後在 PowerShell 輸入：

```powershell
python --version
```

看到 `Python 3.12.x` 就成功。

---

## 第 3 步：安裝 Node.js（含 npm）

1. 到 Node.js 官網下載 LTS 版  
   `https://nodejs.org/`
2. 一路 Next 安裝預設值
3. 安裝完成後測試：

```powershell
node --version
npm --version
npx --version
```

三個都有版本號就成功。

---

## 第 4 步：安裝 VS Code

1. 到 VS Code 官網下載安裝  
   `https://code.visualstudio.com/`
2. 安裝時可勾選「加入 PATH」與右鍵選單（建議）

---

## 第 5 步：在 VS Code 安裝 Continue 擴充套件

1. 開啟 VS Code
2. 左側點「Extensions（擴充套件）」
3. 搜尋 `Continue`
4. 安裝官方 Continue 擴充套件

---

## 第 6 步：安裝 LM Studio

1. 到 LM Studio 官網下載  
   `https://lmstudio.ai/`
2. 安裝完成後開啟 LM Studio

---

## 第 7 步：下載本專案

開 PowerShell，切到你要放專案的路徑，執行：

```powershell
git clone https://github.com/lioil1020-JackLee/local-coding-agent.git
cd local-coding-agent
```

---

## 第 8 步：安裝 uv 與專案依賴

## 8.1 安裝 uv

在 PowerShell 執行：

```powershell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

安裝後關掉 PowerShell 再開一個新視窗，測試：

```powershell
uv --version
```

## 8.2 安裝本專案依賴

在專案根目錄執行：

```powershell
uv sync --extra local
```

說明：

1. 本專案以 `pyproject.toml` 為主
2. `requirements.txt` 是補充參考，不是主安裝流程

---

## 第 9 步：準備 Continue 設定資料夾

你有兩種用法，選一個即可。

## 用法 A（新手推薦，最簡單）

直接用預設路徑：

`C:\Users\<你的帳號>\.continue`

把專案內 `continue/config.yaml` 複製到這裡：

`C:\Users\<你的帳號>\.continue\config.yaml`

## 用法 B（你目前慣用 F 槽）

1. 先建 `F:\.continue`
2. 建 Junction（把預設位置指到 F）

```powershell
cmd /c mklink /J "C:\Users\<你的帳號>\.continue" "F:\.continue"
```

---

## 第 10 步：設定 API 金鑰環境變數

你需要一個 key：

1. `GITHUB_TOKEN`（GitHub MCP）

## 10.1 建立 GitHub token

1. 到 GitHub 設定頁建立 Personal Access Token
2. 給最小必要權限（至少可讀你要查的 repo）

## 10.2 寫入 Windows 環境變數（永久）

```powershell
setx GITHUB_TOKEN "你的 GitHub Token"
```

設定後一定要：

1. 關掉所有 VS Code 視窗
2. 再重新開啟 VS Code

---

## 第 11 步：在 LM Studio 下載與載入模型

## 11.1 先下載主力模型（穩定）

建議先用：

1. 模型：`qwen/qwen2.5-coder-14b-instruct`
2. 作者：`lmstudio-community`（優先）

## 11.2 再下載進階模型（可選）

1. 模型：`qwen/qwen3-coder-30b-a3b-instruct`
2. 作者優先順序：
   - `lmstudio-community`
   - `unsloth`
3. 量化建議（16GB 顯卡）：
   - 首選：`Q3_K_L`
   - 次選：`Q4_K_M`（可能較慢）

## 11.3 LM Studio 推論參數建議

### qwen2.5-14b

1. `contextLength / num_ctx`: 20480
2. `maxTokens`: 3072

### qwen3-30b

1. `contextLength / num_ctx`: 12288
2. `maxTokens`: 2048
3. 若顯存壓力大，降到 `8192`

## 11.4 啟用本地 API

在 LM Studio 啟動 Local Server，確認 API Base 是：

`http://localhost:1234/v1`

---

## 第 12 步：檢查專案設定是否正確

打開本專案：

`continue/config.yaml`

確認有這些 MCP：

1. `filesystem`
2. `github`
3. `ddg-search`
4. `repo_guardian`

---

## 第 13 步：正式驗證（一定要做）

在專案根目錄跑：

```powershell
uv run pytest -q
npx -y @oevortex/ddg_search@1.2.2 --help
```

預期結果：

1. 測試全綠
2. DDG MCP 可啟動

---

## 第 14 步：Continue 實際測試

開 VS Code 的 Continue 聊天視窗，先試三句：

1. 「請用白話告訴我這個專案在做什麼」
2. 「請查今天台北天氣並附來源」
3. 「請幫我安全修改 README，一次只改一小段」

如果都正常，就表示整套成功了。

## 常見錯誤與解法

## 1) `DDG MCP 啟動失敗`

解法：

1. 完整重開 VS Code
2. 再測 `npx -y @oevortex/ddg_search@1.2.2 --help`

## 2) Continue 看不到你更新的 config

解法：

1. 確認實際路徑是 `C:\Users\<你>\.continue` 還是 `F:\.continue`
2. 若有 Junction，兩者其實是同一份
3. 重開 VS Code

## 3) 模型回覆很慢或常爆記憶體

解法：

1. 先用 qwen2.5-14b
2. 降低 context（例如 20480 -> 12288 -> 8192）
3. 改用較輕量化版本（例如 Q3_K_L）

## 4) npx 下載失敗

解法：

1. 確認網路可連 npm
2. 重試同一指令
3. 檢查防火牆/代理設定

## 搬到新電腦時的重點

1. 程式碼可直接 git clone
2. 金鑰一定在新機重新設定（不要直接貼在檔案）
3. `agent_runtime` 不用硬搬，讓新機自動重建
4. 最後一定跑一次「第 13 步正式驗證」
