# 目標與方向

本文件描述本地端程式編輯代理（Local Coding Agent）的整體目標、策略與發展路線，以期逐步達到 Cursor‑like 的體驗。

## 計畫進度

- Phase 1 – 安全編輯管線 ✅  
- Phase 2 – 執行控制 ✅  
- Phase 3 – 執行接管 ✅（已實作 `run_task_pipeline`、`propose_patch`、`stage_patch`）  
- Phase 4 – 代理核心開發（進行中）  
- Phase 5 – 端到端整合（計劃中）  

## 目標

打造一個具備安全、決定性且可追溯執行能力的程式編輯代理，讓使用者能用自然語言完成程式分析與修改，並支援多檔案變更、複雜 refactor、驗證與回滾等功能，體驗盡量接近 Cursor。  

## 策略與方向

實現此目標需要以下策略：

- **決定性執行**：使用 `ExecutionController` 管理所有步驟，支援 retry、stop 與 fallback，確保相同輸入產生相同結果。  
- **統一入口**：`run_task_pipeline` 依 `task_type` 自動選擇分析或編輯流程，簡化使用者指令。  
- **Session 隔離**：所有寫入操作在 sandbox worktree 執行，並透過 diff 預覽與驗證機制避免汙染主線。  
- **規劃與補丁產生**：Planner 解析自然語言指令並產生步驟，Patch Generator 透過語言模型生成補丁，Validation Controller 確認修改安全。  
- **對話記憶**：代理保留 conversation_id 與 session_id，用於多步任務與後續查詢。  
- **多輪規劃與高階操作**：規劃支援 rename symbol、extract function、move file 等高階 refactor 工具，並由 Planner 自動組合步驟。  
- **模型驅動的 patch 產生**：整合語言模型生成跨檔案、多範圍的補丁與重構，遵守 patch policy 避免插入無關內容。  
- **多層架構**：從 UI → Agent → MCP tools → Services → Sandbox，分層處理分析、規劃、執行與驗證，方便維護與擴充。  

## 路線圖

### Phase 4 – 代理核心開發（進行中）

- 建立穩健的 Planner：解析白話指令並結合程式碼分析、向量搜尋與對話記憶，產生細緻的多步計劃。  
- 整合補丁產生器：透過語言模型產生跨檔案 patch，支援 rename symbol、extract function 等 refactor 操作，並遵循 patch policy。  
- 擴充驗證控制器：加入 flake8、mypy、black、pytest 以及自訂規則，提供語法、型別與行為檢查。  
- 開發對話記憶與多輪互動：保存 conversation_id、session_id 與上下文，支援追蹤先前修改與提問。  

### Phase 5 – 端到端整合（規劃中）

- 開發高階 refactor 工具：如檔案搬移、重新命名符號、擷取函式/類別、重新排序引數等，並整合到 MCP 層與 Planner。  
- 強化 MCP Server：支援 API 金鑰認證、日誌、效能監控、非同步與串流，以及 CI/CD。  
- 完整整合 Continue 與其他前端：提供完整的 `.continue/config.json` 設定，支援互動式 diff 預覽、回滾與進度展示。  
- 建立 CI/CD：結合 pytest、flake8、mypy、black 等工具與分支保護規則，確保品質與安全。
