# 變更紀錄與開發進度

本檔案彙整了專案的變更歷史與當前開發進度，方便快速了解已完成的功能與未來規劃。

## 變更紀錄

### 最新變更

以下為近期整合的功能與改進：

- 統整重試守護與回退政策至 ExecutionController 與編輯管線，使流程更穩定。  
- 新增 `run_task_pipeline_tool`，統一分析與編輯流程。  
- 加入 `propose_patch`、`preview_diff`、`stage_patch`、`move_file` 等工具，支持生成與套用補丁及檔案移動。  
- 新增 MCP server 骨架，支援 API 金鑰認證與日誌紀錄。  
- 新增代理層：包含 Planner、Patch Generator 及 Validation Controller 的初步實作。  
- 驗證鉤子更新：加入靜態檢查與行長限制，強化安全性。  
- 新增對話記憶機制並更新 Continue 設定檔。  

### 未來規劃

未來將持續朝 Cursor‑like 體驗邁進，規劃包含：

- 發展更智慧的自然語言規劃器，理解模糊指令並依據對話歷史規劃多步驟任務。  
- 深度整合補丁產生器與語言模型，支援跨檔案修改與重構，自動補充缺失 context。  
- 擴充驗證機制：加入 flake8、mypy、pytest 等靜態與動態分析，並支援使用者自訂規則。  
- 實作高階重構工具，例如 rename symbol、extract function、move file 等，並於 MCP 層註冊。  
- 加強 conversation 記憶與多輪互動，管理 conversation_id 與 session_id，以支援長期任務。  
- 強化 MCP server：提供非同步處理、效能監控、CI/CD 介接與 API 金鑰管理。  
- 完整整合 Continue 外掛與 VSCode，提供豐富的工具面板與互動式 diff 預覽。  

### 歷史記錄

較早期的里程碑：

- 引入 ExecutionController 管理流程。  
- 修復相容性問題。  
- 所有單元測試（pytest）通過。
