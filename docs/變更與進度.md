## 現在的關鍵定位更新
到這一輪文件更新為止，專案定位已正式升級。

### 舊理解
- 安全修改 backend
- tools + pipeline
- Continue 整合導向

### 新理解
- **本地端 CLI Agent**
- **具備 Skill 系統**
- **safe-by-default**
- **以 Cursor Agent 等級能力為明確對標**

這不表示前面的 backend/pipeline 工作不重要。  
相反地，它們現在應被理解為：

> **通往完整 CLI agent 的已落地主線基礎。**

## 目前專案處於哪個階段
若以終極目標來看，專案現在大致位於：

> **「已具備安全修改 runtime 與 session/lifecycle 基礎，正準備從穩定 pipeline/backend 進入完整 CLI agent + skill system 的建設期」**

也就是說：

- 不是從零開始
- 不是 demo
- 也還不是完整 Cursor 級 agent
- 而是正在從可靠 backend 主線升級成真正 agent 產品

---

## 目前最建議的下一步
最合理的順序：

1. 將 session lifecycle 行為更完整收斂到 orchestration 入口
2. 減少 session metadata touch/update 的重複
3. 建立 planner + executor agent loop
4. 導入 skill abstraction / registry / routing
5. 強化 CLI product surface
6. 最後再深化 Continue / IDE integration
